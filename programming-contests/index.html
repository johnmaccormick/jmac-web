Information about programming contests

  • How does a programming contest work?: You are in a team of up to three people. You have only one computer. You are given a bunch of problems to work on. You submit your solutions (Java, C, or C++ programs) to an online judging system. You can submit multiple times. You score points for correct solutions, and lose points for incorrect solutions. See http://icpc.baylor.edu/worldfinals/rules and acm-regional-2012.pdf for examples of the detailed rules of particular contests. The second link here also has a couple of example problems.
  • How can I practice?: Just search for online problem sets and solutions. To build confidence, do a few high school level problems (e.g. UMD Programming Contest). Then you could tackle some real problems from the ACM repository, which has an online judging system.
  • What kind of input and output do the problems use? Typically, your programs have to read input from standard input and send output to standard output. In Java, this is System.in and System.out respectively. Most inputs can be read easily using the Scanner class.
