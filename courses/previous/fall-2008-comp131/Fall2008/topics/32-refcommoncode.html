Practice Exercises

Use the StudentDatabase-v1 project to complete each of the following exercises:

  1. Add a changeInstructor method to the Student class. This method takes the department and number of the class and the name of the new instructor as parameters, and changes the instructor of the class. If the student is not taking the specified class, the method should print an error message Be sure to include a javadoc comment for your method.

    Hint: you should not have to write a loop to implement this method!

  2. Write test cases for the changeInstructor method that properly test it (statement coverage is a minimum goal).
  3. Add the following two methods to the Student class:
    • a method called isTakingFrom that takes the name of an instructor as a parameter and returns boolean - true if the student is taking a course from that instructor, and false otherwise.
    • a method called dropCourseOf that takes the name of an instructor as a parameter, and drops one course from the student's schedule that the specified instructor is teaching.
    These methods should not contain any repeated functionality. Be sure to include javadoc comments for these methods.
  4. Write test cases for the isTakingFrom and dropCourseOf methods that properly test these methods.

Homework Assignment 10a

Download and unpack the LibraryDB project and use it to complete the following exercises.

  1. Study the interface view for the Book class, as you will need to use instances and methods of this class in completing the following exercises. The implementation of the Book class and test cases achieve statement coverage have been provided for you. (There is nothing to turn in for this exercise.)
  2. Complete the definitions of the constructor and the getLibraryName, getNumBooks, addBook and getBook methods of the Library class.
  3. Create a test class for the Library class, and write a test case for the constructor.
  4. Write test cases for the addBook and getBook methods that properly test those methods. In particular, you should test that getBook behaves correctly when returning a book at the beginning, middle and end of the collection, and also when the library does not have a book with the specified call number.
  5. Write test cases for the hasBookByAuthor and deleteBookByAuthor methods (that have been provided for you) that properly test those methods. Statement coverage is a minimum goal. In particular, you should test that these methods behave correctly when a book by the specified author is at the beginning, middle and end of the collection, and when the library has no books by the specified author.
  6. The hasBookByAuthor and deleteBookByAuthor methods contain repeated functionality for finding the position of a Book by a particular author. To eliminate this repetition, refactor the methods as follows:
    • add a new method that gets the position of a Book by a particular author (specified as a parameter), or indicates that no Book by this author was found in the library.
    • use your new method to refactor the hasBookByAuthor and deleteBookByAuthor methods so that they no longer contain any loops
    Make sure that your test cases still pass after this refactoring.
