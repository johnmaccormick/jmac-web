Practice Exercises

  1. Draw an object diagram for instances of the BankCustomer and Account classes representing yourself and your bank accounts. (Please do NOT use your actual bank account numbers and balances.)
  2. Open the MyComputer project that you completed as a homework exercise in Class #10, create a new class ComputerOwner within that project, and implement the ComputerOwner class as follows:
    1. Declare fields so that an instance of class ComputerOwner can hold the following information for a computer owner: name, address, telephone number and a reference to a Computer object representing the computer that the person owns.
    2. Define a constructor that initializes all fields of a ComputerOwner using parameters.
    3. Define accessor methods for the name, address and telephone number.
    4. Create a test class for ComputerOwner, and write a test method for the constructor. Your test method should not check the value of the Computer object associated with a ComputerOwner.
  3. Do Exercises 3.1, 3.4, and 3.22 - 3.25 in Barnes & Kolling. Many of these exercises use the clock-display project from Chapter 3.

Homework Assignment 6b

  1. Download and unpack the Bicycle project, open it in BlueJ, and use it to complete the following steps.
    1. complete the definitions of the BicycleSeat and Tire classes according to the specifications given in those classes
    2. create test classes for the BicycleSeat and Tire classes, and write test cases that achieve statement coverage for those classes
    3. (partially) complete the definition of the Bicycle class according to the specifications given in that class. Note that you should NOT complete all of the methods for this homework assignment - you should stop at the comment that says: 
      // STOP HERE FOR CLASS #18 HOMEWORK
    4. create a test class for the Bicycle class, and write a test case that checks that the constructor sets the manufacturer and number of gears correctly for a Bicycle object. You do not need to check the values of any other fields set by the constructor.
    5. you will be submitting your Bicycle project via WebCAT as part of a future homework assignment, so there is nothing to submit for this exercise (yet).
  2. Using the Bicycle project that you worked on in the previous exercise:
    1. Draw a class diagram for the classes in this project.
    2. Draw an object diagram for an instance of the Bicycle class. Use any values for the fields that are of the appropriate type and that could reasonably represent a real bicycle.
