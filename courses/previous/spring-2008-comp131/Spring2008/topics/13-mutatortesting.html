Practice Exercises

  1. Create a test method for the insertMoney method in the TicketMachine class from the naive-ticket-machine project from chapter 2 of Barnes & Kolling.
  2. Add a method named discount to the TicketMachine class. This method does not return a value and accepts one integer parameter. Its parameter indicates a discount that should be applied to the price of all future ticket purchases (i.e. the price of a ticket should be reduced by the amount of the discount). Be sure to include a JavaDoc comment for your discount method. Copy and paste the source code for your discount method as your solution to this question.
  3. Create a test method for the discount method that you just added to the TicketMachine. Copy and paste the test method that BlueJ generated as your solution for this question.
  4. Create a new method named empty in the TicketMachine class. This method does not have any parameters and does not return a value. When called it removes all of the money from the machine (i.e. it set the machine's balance to 0). Be sure to include a JavaDoc comment for your empty method. Copy and paste the source code for your empty method as your solution to this question.
  5. Create a test method for the empty method that you just added to the TicketMachine. Copy and paste the test method that BlueJ generated as your solution for this question.

Homework Assignments

  1. Add a method named getDollars to the Account class. This method does not accept any parameters and returns an integer indicating the number of whole dollars there are in the account. For example, if the balance of the account is 754 cents, the a call to getDollars will return 7. Be sure to include a JavaDoc comment for your getDollars method. Copy and paste the source code for your getDollars method as your solution to this question.
  2. Create two test methods for your getDollars method. One case should test when there are 0 dollars and one where there is at least 1 dollar. Copy and paste the test methods that BlueJ generated as your solution for this question.
  3. Add a method named addInterest to the Account class. This method does not return a value and accepts no parameters. A call to this method should increase the balance of the account according to the interest rate. For example, if the balance is 200 and the interest rate is 20, then 200 * 20 / 100 gives the number of cents to increase the balance by. Use a local variable to hold the amount that the balance should increase by. Be sure to include a JavaDoc comment for your addInterest method. Copy and paste the source code for your addInterest method as your solution to this question.
  4. Create a test method for your addInterest method. Copy and paste the test method that BlueJ generated as your solution for this question.
  5. Add a method named jump to the Turtle class. This method does not return a value and accepts two double values as parameters. The first parameter specifies a new x position for the turtle and the second parameter specifies a new y position for the turtle. The result of calling this method is that the turtle's x and y positions are changed to the specified value - i.e. the turtle "jumps" to the new position. Be sure to include a JavaDoc comment for your jump method. Copy and paste the source code for your jump method as your solution to this question.
  6. Create a test method for the jump method that you just added to the Turtle class. Copy and paste the test method that BlueJ generated as your solution for this question.
