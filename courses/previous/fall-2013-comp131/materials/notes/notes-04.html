Topic #4     K&S With Assembly Language Programming

 

Next lab will be writing ASM (Assembly language) programs for K&S with control unit

 

K&S has many of the main elements of the stored program architecture.

CPU = control unit, ALU, Registers

Memory

No I/O devices, but everything else.

 

But as is the K&S is not quite a true stored program architecture yet.

         Program and data are stored in memory!

        

         Final version of the K&S with Assembly Language

         This one is a true stored program architecture machine.

 

         ALU, Registers, Main Memory are all the same

         Control unit has changed a little.  The big difference is that we’ll now store          program instructions in main memory.

 

                  We’ll be starting to write our programs in Assembly Language

                  instead of micro-machine language or machine language

 

Assembly language is a slightly more human readable language for writing programs.  The best way to understand it is to see an example:

 

         Example:

                  R3 = R0 + R1 – R2     (Our goal)

 

                  Break into 2 steps:

                           R3 <- R0 + R1

                           R3 <- R3 – R2

 

         The Assembly Language program for this: (second column)

                 

                  0:      ADD R3 R0 R1         

                  1:      SUB R3 R3 R2          

                  2:      HALT      

                 

                  Still a little cryptic but much better than 1’s and 0’s!

 

Main thing to be aware of at this point is that program instructions will run one after the other in the order that they are written.  This is something we call Sequential Execution and it is the default mode for executing program instructions – just one after the other in the order they appear in the computer’s memory.

        

         We can enter (type) these instructions directly into the computer’s memory.       

                  Note the change from “Auto” to “Inst”

 

When we run this program, notice these two elements of the control unit and see if you can figure out what they do:

 

                  PC = Program Counter

IR = Instruction Register

                          

                           The IR holds the instruction that is currently being executed.

                           The PC holds the address (location) of the memory cell that holds                                 the next instruction to be executed.  It is what controls the                                         sequential execution.

 

                  Reset… sets PC to 0 so that program starts over.

        

The Instruction Interpretation unit does a lot of the work of executing the assembly language instructions.

                  It translates the instruction in the IR into Knob and Switch settings.

 

Another Example with both data and instructions in memory:

                  MM[10] = MM[8] + MM[9]              

 

                           R0 <- MM[8]                     

                           R1 <- MM[9]

                           R2 <- R0 + R1

                           MM[10] <- R2

 

LOAD and STORE.

 

                  0:      LOAD R0 8                                 

                  1:      LOAD R1 9

                  2:      ADD R2 R0 R1

3:      STORE 10 R2

                  4:      HALT

 

         Enter data into MM[8], MM[9], MM[10].

        

These instructions are nothing special.  Essentially what we have done is given easier to remember names to the common patterns of 1’s and 0’s and then built some hardware to do the translation for us.  The whole idea being to make the computer easier for humans to program.

 

This is good, but...

         In what we’ve seen so far every program runs sequentially

         And it runs the same way every time.

 

         Some programs will have to make decisions.

Allowing them to do different things at different times.

To achieve this, there are some special instructions that do not perform sequential execution and instead branch (jump) to a different location in memory -- to an instruction that is not the next one in the sequential execution.

 

For example:

If the sum of R0 and R1 is zero then MM[15] <- R0

otherwise MM[15]=R0+R1.

 

0:      ADD R2 R0 R1

1:      BZERO 4

2:      STORE 15 R2

3:      HALT

4:      STORE 15 R0

                           5:      HALT

 

         BZERO – change the PC to 4 if the last ALU operation had a zero result.

 

                  If branch is not taken, then sequential execution continues!

 

See all of the Assembly language instructions and the description of what they do.

 

 

                  If R0 is negative then R1 <- MM[10]

                  Otherwise R1 <- MM[11]

 

                  0:      ADD R0 R0 R0

1:      BNEG 4

                  2:      LOAD R1 10

                  3:      HALT

                  4:      LOAD R1 11

                  5:      HALT

 

Notice that we must do an ALU operation just before the branch, in order   for the branch to work.

 

Note that the micro-machine language that we have seen is equivalent to the       assembly language.  The actual machine language is just slightly different     from the micro-machine language – some groups of bits are represented in      coded form.
