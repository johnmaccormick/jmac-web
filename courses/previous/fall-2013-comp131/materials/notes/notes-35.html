Finding and Reading Javadoc Documentation

 

To fix the print and printStandingStats methods, we need to be able to iterate over all of the Students.

However, a HashMap does not allow us to use a for loop as an ArrayList did.

 

To figure out how to we can iterate over all of the elements in a HashMap we’ll look at the documentation for the HashMap class!

Sun, creators of Java also provide javadoc for every class that comes with Java.  So we can learn about all of the provided classes in the same way we have been learning about our classes by looking at the interface view.

 

To see the documentation for the built in classes in BlueJ we need to:

 

Choose:

                  Help -> Java Class Libraries

 

This will bring up a web browser containing the documentation

The top left frame lists all library packages (including java.util)

The bottom left frame lists all classes in those packages

The larger right frame shows the details of a selected class

 

Click java.util in the upper left frame,

 

Find ArrayList and look at it first.

It implements Collection.  as the key to knowing whether or not you can use a for each loop.

 

Then find HashMap in the lower left frame and click it

Scan through the constructor and method summaries

 

Note that the size() method in fact returns the number of key, value pairs in the HashMap.

 

The keySet method looks promising

If we had all the keys, we could retrieve all of the students

 

But what is a Set? (the return type from keySet)

   Click on Set to find out (open in a new tab or window)

Writing javadoc documentation:

The documentation for a class should allow other programmers to use it correctly WITHOUT looking at the implementation of the class.  It allows them to use abstraction!

 

At a minimum, documentation for a class should include:

The class name

A description of the overall purpose of the class

A version number or date

Authors

Documentation for each public constructor, method and constant

         (Why not documentation for private methods and fields?)

 

The documentation for each method and constructor should include

A description of its purpose

The meaning of each formal parameter

A description of the return value (if any)

 

The documentation can be generated by choosing the interface view in BlueJ

Look at the interface view for the RegistrarsOffice and Student classes

 

Note that the pieces mentioned above are there, and the correspondence between the javadoc comments and the contents of the interface view

 

Note that it looks exactly like the "official" documentation for HashMap etc.
