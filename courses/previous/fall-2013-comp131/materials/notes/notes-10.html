non-void return type and return Statement

Methods can also “print” information, causing it to be displayed in a Terminal Window.    The print method in the Account class is one such method.

 

         The getBalance method uses a return statement to return its value.

         The print method uses System.out.println to display its output on the terminal.

                  System.out is an object built into Java.

                  println is a method that can be invoked on that object.

Invoking the println method causes the actual parameter to be printed to the terminal window.

 

+   is a concatenation operator (when used between Strings).

And if one of the operands is a String and the other isn’t, then a copy of the value of the one that isn’t a string is first converted to a string and then used in the concatenation.

 

 

                  "Account number: " + acctNumber

 

Assuming that acctNumber is of type int, this causes the value of the acctNumber field (copied and converted to a string value) to be concatenated onto the end of the String “Account Number” and the resulting String is used as the actual parameter to the println method.

 

                  Similarly for the second line:

 

"Balance is: " + balance + " cents."

 

The value of the balance field to be concatenated onto the end of the String “Balance is” producing “Balance is 567”.  The second + then causes the String “ cents.” to be concatenated onto the end of that String and the whole thing is then used as the actual parameter to the println method.

 

So notice that + has two meanings.  Addition and concatenation.  You should be able to tell the difference between the two by the context in which it is used.

 

         567 + 321                       (addition)

                  “testing “ + 123     (concatenation)

                  balance + 7                           (addition)

                  “345” + “876”             (concatenation)

                                             “balance” + “9”        (concatenation)

 

Let’s make an improvement to the Account class.

It would be nice if instead of printing out the balance of the account in cents it were printed out in a more natural way such as in dollars and cents.

 

There are 2 arithmetic operations in Java that will help us with this task:

         Integer Division which is implemented using the / operator

         Modulo (remainder) which is implemented using the % operator

 

In Java when two int values are divided the result is also an int:

E.g.   7/2 is 3                    // truncated = fractional parts are lost

         9/4 is ???

         567 / 100 is ???

 

In Java the % operator computes the remainder of a division.

         E.g.   7 % 2 is 1.          // Only remainder is kept

                  9 % 4 is ???

9 % 5 is ???

10 % 5 is ???

567 % 100 is ???

 

 

So, we can convert cents to dollars and cents.

         Modify the print method:

System.out.println("Balance is: $" + (balance/100) +  “.”  + (balance%100) + ".");

 

         There is, however, a problem with this code!

What would the output be if we have a balance of say 307.

         Balance is: $3.7.

 

This can be fixed by just using / and % operations, but we will learn a better way to format output later on.
