public vs. private

 

Question: Would code outside of the Student class ever need to use the findCoursePosition method that we developed in the last class?

 

No!  We only added it to improve the implementation.

 

But because findCoursePosition is declared as public it still appears in javadoc documentation (demo) and it can still be called by code outside of the class.  This clutters the "interface" of the class - it is not clear to other programmers that they don't need to use the findCoursePosition method

 

The solution to these problems is to declare the findCoursePosition method as private!   It disappears from the javadoc

                 

public: field or method is visible to code in other classes.

Available for external use

Code in other classes can use the field or method directly.

 

private: field or method is NOT visible to code in other classes.

         For internal use only

External code can only use the field or method indirectly.

         E.g.    Calling an accessor or mutator method uses the fields indirectly.

                 

The use of public and private are related to an idea called information hiding.

The idea behind information hiding is that the programmer of a class is able to hide the details of how a class works from someone who is using the class.

For example:

A user of our Student class (programmer of client code) would not be able to tell whether we used an array or an ArrayList to hold the Courses in the schedule.

 

Similarly, they could not tell if the isTaking and dropCourse methods were written the way we had them originally or the way we rewrote them.

 

With abstraction we can use a class without worrying about its details.  This is only possible if the class has sufficiently hidden its implementation from us via information hiding.

 

Information hiding forces users of a class to rely only on the public interface, i.e. to use them as abstractions. Thus, it becomes possible to refactor classes without breaking anyone else’s program.  This is a huge benefit in software engineering.

 

General rules of thumb:

All fields are declared private

Methods for outside use are declared public

Methods for internal use are declared private.

Let’s look at two methods: dropCourse and enrollCourse.

         Each takes a stuID as a parameter.

dropCourse takes the department and number of the course to drop from the student’s schedule

         enrollCourse takes a course to add to the student’s schedule.

 

         If we think about those two methods they are going to share some functionality.

                  What is it?

They both have to iterate through the list of students to find the student with the specified ID number.

dropCourse then drops the course from that student’s schedule.

enrollCourse then adds that course to the student’s schedule.

 

So we could either repeat the code for finding the student in both of those methods, or we could write a method that finds a student by student ID and returns it.  Then use that method in both dropCourse and enrollCourse.

 

From what we learned last class, it should be clear that the second option is the way to go. 

         So let’s write a method that gets a student by ID.

 

 

 

    /**

     * get a student by student ID

     *

     * @param stuID the student ID

     * @return the student with the specified student ID, or null if no

     * student has that student ID

     */

    public Student getStudent(String stuID) {

       int stu=0;

        Student found = null;

        while (stu < students.size() && found == null) {

          current = students.get(stu);

if (stuID.equals(current.getStudentID())) {

                found = current;

            }

          else {

              stu++;

          }

        }

        return found;

    }

 

This method may be useful to code in other classes, so it can be declared public.

     * @param number the number of the course to drop

        }

    }

First, we add a method to the Student class to return the standing of the student given their graduation year and the current year

                  (write getStanding method in Student)

        

     /**

      * return the standing (senior, junior, ...) of the student given the

      * current year

      * @param year the current year

      */

     public String getStanding(int year) {

         int yearsLeft = graduationYear - year;

         if (yearsLeft == 0) {

             return "senior";

         } else if (yearsLeft == 1) {

             return "junior";

         } else if (yearsLeft == 2) {

             return "sophomore";

         } else if (yearsLeft == 3) {

             return "first year";

         } else {

             return "alumnus";

         }

     }

    /**

     * print the number of students of each standing

     * @param year the current year

     */

    public void printStandingStats(int year) {

        int seniors = 0;

        int juniors = 0;

        int sophs = 0;

        int fstyrs = 0;

        int alums = 0;

       

  for (int s=0; s < students.size(); s++) {

            Student current = students.get(s);

            String standing = current.getStanding(year);

 

            if (standing.equals("senior")) {

                seniors++;

            } else if (standing.equals("junior")) {

                juniors++;

            } else if (standing.equals("sophomore")) {

                sophs++;

            } else if (standing.equals("first year")) {

                fstyrs++;           

            } else {

                alums++;           

            }

        }

 

        System.out.println("Number of students by standing: ");

        System.out.println("Seniors: " + seniors);

        System.out.println("Juniors: " + juniors);

        System.out.println("Sophomores: " + sophs);

        System.out.println("First Years: " + fstyrs);

        System.out.println("Alumni: " + alums);

    }
