Topic #25: Modifying Elements & Searching Collections

 

Recap: In the last several classes we saw how to iterate over the elements of a collection:

            E.g.     printAllDVDs

                        countRentalNights

 

Sometimes it might be useful to do something to either all of the elements of a collection or to the elements of the collection that meet some criterion:

 

For example, every night at midnight it might be useful to have a method that goes through the collection and increases the number of rental nights of each checked out DVD by 1.

 

    /**

* Increase the number of rental nights on each rented

* DVD by 1 night.

*/

public void increaseRentalNights() {

     // For each DVD in the store.

         // if it is checked out.

              // increase its rental nights.

}

 

Translate that into java

 

    /**

* Increase the number of rental nights on each rented

* DVD by 1 night.

*/

public void increaseRentalNights() {

     // For each DVD in the store.

     for (int i=0; i<dvdList.size(); i++) {

         DVD aDVD = dvdList.get(i);

         // if it is checked out.

         if (aDVD.getNightsRented() > 0) {

              // increase its rental nights.

              aDVD.addRentalNight();

         }

     }

}

 

Trace this and inspect.

 

When a method modifies the elements of a collection we will want to be a little more careful when testing it.  Just making sure that we have statement coverage is not really sufficient for checking that the method works correctly. So we have a testing rule for this type of method:

 

Testing Rule for Methods that modify the elements of a Collection:

Modify the elements collection

Check at least one modified element

Check at least one unmodified element (if any)

Check just enough fields

With collections it is often going to be useful to search through the collection for objects with particular properties:

            This will allow us to do things like:

            E.g.     Find the DVD that has been rented for the longest period of time.

                        Find the most overdue DVD.

 

           

We need a method that searches for the DVD that has been rented for the longest period of time.  This method should return a reference to the DVD.

           

What would the method signature look like?

 

            public DVD getLongestRental() {

}

 

            Add a javadoc comment.

 

     /**

      * Find and return the DVD that has been rented

      * for the longest period of time.

      */

 

What’s the plan for implementing this method?

The basic idea will be to iterate through the DVDs keeping track of the longest rental that we have seen so far.

 

            To start assume that the first DVD would be the longest rental so far.  It is, as we start !

           

Check each DVD to see if it has been rented longer than the longest one we’ve seen so far.

If it is then make it the longest rental we’ve seen so far.

 

Once we’ve been through all of the DVDs in the collection we will have found the longest rental.

Sketch out the code:

 

            public DVD getLongestRental() {

 

         // Set the first DVD as the longest rental so far.

         // Iterate through the DVDs.

              // If the current DVD has been rented longer than

              // the longest one seen so far then set it

              // as the longest so far.

 

         // Return the DVD with the longest rental.

}   

 

Fill in the details.

 

public DVD getLongestRental() {

 

     // Set the first DVD as the longest rental so far.

     DVD longest = dvdList.get(0);

 

     // Iterate through the rest of the DVDs.

     for (int i=1; i<dvdList.size(); i++) {

          DVD aDVD = dvdList.get(i);

          // If the current DVD has been rented longer than

          // the longest one seen so far then set it

          // as the longest so far.

          if (aDVD.getNightsRented() > longest.getNightsRented()) {

               longest = aDVD;

          }

          else {

               // do nothing...

          }

}

 

     // Return the DVD with the longest rental.

     return longest;

}

 

It is possible to use if without else, so remove.

            There is also another problem, what is it?

                        This method returns the first one!

                        How could we fix it?

Add another if statement at the end that checks if the DVD being returned has been checked out.  If it has not, then return null instead.

 

Test methods necessary to achieve statement coverage for getLongestRental.

            Test on an empty collection

            Test on non-empty collection with at least one rental

 

Note that if the longest rental is the first one, we do not achieve statement coverage.

                                    Why not?

 

If I wanted to be really thorough with my testing then I would probably want to test that the method works when the longest rental is the first DVD in the collection and when it is somewhere in the middle of the collection and also when it is the last DVD in the collection.

                       

Statement coverage is the minimum level of testing that you should be doing.
