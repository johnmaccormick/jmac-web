So we have a situation here where we need our program to take different actions depending upon some condition.  What should we do in the case of an over draught?

 

So how does it do that… lets look at the code.

         Look at withdraw method in Account-v3.

 

/**

     * withdraw money from the account

     * @param withAmount the amount to withdraw

     */

    public void withdraw(int withAmount) {

        if (balance >= withAmount) {

           balance = balance - withAmount;

        }
        else {

           System.out.println("Error: there are only " +

balance + " cents in this account!");

        }

    }

 

         This code contains a Conditional Statement (a.k.a. if statement)

The way this statement works is as follows: If the condition expressed between the () is true, then the statements in the block of code following the condition are executed. If the condition is false, then the statements in the block of code following the else are executed.

 

 

General Form of a Conditional Statement:

 

         if (<condition>) {

         <Statements to execute if condition is true>

}

else {

         <Statements to execute if condition is false>

}

 

         The <condition> must evaluate to true or false.

 

Notice that each time the if statement is executed, one or the other of the blocks of code will be executed, but never both!

 

Lot’s of situations where we need to put decision making into our program logic.

More about the <condition>:

 

The <condition> is a Boolean Expression:

         An expression that evaluates to either true or false.

         E.g:      amount > 0

         amount <= balance

         balance – amount >= 0

 

Each of these expressions will evaluate to either true or false depending on the values of amount and balance.

 

Are any of these Boolean expressions equivalent to each other?

        

There will be many different ways to write the same condition.  Some will be easier for humans to understand than others.  The computer basically doesn’t care how you write them; so try to write your conditions in a way that makes the most sense to a human reader.

 

Very often Boolean Expressions will involve the use of Relational Operators

 

         <

    <=

    >

    >=

         ==              (== is comparison, = is assignment) 

VERY FREQUENT ERROR!

         !=              (not equal to)

 

Testing with Conditionals:

 

When we include if statements in our programs we need to be a little more careful about testing our code.  We want to make sure that we test both parts of the if.  Or more generally, we will want to achieve something that is called Statement Coverage.

 

         Statement Coverage:

                  Test cases execute every statement in the program at least once.

 

This is the minimum standard for testing that we’ll be using for the rest of the semester. That is the test cases that you create for a class should, at a minimum, cause every statement to execute at least once.

 

Testing the withdraw method:

To achieve statement coverage for the withdraw method we’ll need one test case that executes the if block and one test case that executes the else block.
