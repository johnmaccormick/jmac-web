Recap: Nested and cascading if statements

 

Every if statement contains a Boolean Expression

         What was that?  Expression that evaluates to either true or false.

 

So far we have used the Relational Operators to create Boolean expressions.

         What were they?

                  >, >=, <, <=, ==, !=

 

Today we’ll be looking at the Boolean Operators that allow us to build more complex Boolean expressions (sometimes simplifying the logic of our programs).

 

Boolean (Logic) Operators

         &&   And

         ||        Or

         !        Not

 

Examples:

         (x > 3) && (x < 10)       

Result of an && is true only if both parts are true.

                    What is a value that makes it true?

                    What is a value that makes it false?
                    What are all of the values that make this true?


 

         (x == 2) || (x == 3)

                  Result of an || is true if either one or both parts are true.

                  What is a value that makes it true?

                  What is a value that makes it false?
                  What are all of the values that make this false?


 

         !(x == 2)

                  Result of a! is true if the expression   a  is false.

                  So this expression will be true when x is not 2.

                  Could we rewrite it just using a relational operator?

                           x != 2

 

         !(x < 3)

                  When is this expression true?

                  How could we rewrite it?


                           x >= 3

 

Any expression involving a not operation can be rewritten without it, but sometimes it may make more sense and be easier to understand with the not included.

 

Summary of Boolean operators:

         P && Q      true only if P is true and Q is true

         P || Q      true if P is true, or if Q is true, or if both are true.

         !P             true if P is false

 

How would I write a Boolean expression that is true if a variable x is between 1 and 10 inclusive?

 

         (x >= 1) && (x <= 10)

 

         A common mistake here is to use || in place of &&.

Would || work in place of &&?

                  No! Why not?

 

         If I wanted to use || what would I have to write?

                  (x == 1) || (x == 2) || ...  || (x == 10)

                          

         NOTE: It is not valid to write:    x>=1 && <=10

                  The computer is stupid!

It doesn’t know what you want to compare to 10.

 

Another common mistake:  1<=x<=10   (Algebra yes; Java no.)

Example #2:

Leap Year calculator:

         A year is a leap year if:

It is divisible by 4,

as long as it is not divisible by 100,

unless it is also divisible by 400.

 

                  ( Blame the pope... It was Pope Gregory the 13th’s idea!)

                  (Or God, b.c. the Earth doesn’t orbit the sun at quite the right speed!)

                 

                  Some examples to clarify:

                           2006: No – not divisible by 4.

                           2004: Yes – divisible by 4 and not by 100

                           2000: Yes – divisible by 4 and 100, but also divisible by 400.

                           1900:  No – divisible by 4 and 100, but not by 400.

        

                  So to rephrase in order for a year to be a leap year it must be:

Divisible by 4 AND not by 100

OR

it is divisible by 400.

 

         Overview of Date class:

                  3 fields representing a date

                           day of the month

                           month the name of the month

                           year

 

                  1 constructor  that initializes the fields to specified values

                  print  method that displays the date.

 

isLeap   method that returns true if the date occurs in a leap year and false otherwise.

 

         Code skeleton:

 

if (the year is a leap year) {

         leap = true;

     }

     else {

         leap = false;

     }

 

What is the condition for the if statement?

 

(year % 4 == 0 && year % 100 != 0) || year % 400 == 0

 

Check it,

                 

                  2006: year % 4 == 0 false

                           year % 100 != 0      true

                           year % 400 == 0      false

 

                                    false && true || false ->      false

 

                  2000: year % 4 == 0          true

                           year % 100 != 0      false

                           year % 400 == 0      true

 

                                    true && false || true   ->      true

 

A note on the ()...

         By default && operations are performed before || operations

                           In the same way that * or / is performed before + or –

           So () are not needed here but they make the code more clear.
