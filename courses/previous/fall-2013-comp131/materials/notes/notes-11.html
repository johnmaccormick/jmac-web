Why are the accessors needed?

 

This process is part of the design phase of any software development project.

 

The result of this design is usually an Interface Specification for a class.

 

Look at Turtle interface = documentation view

 

         Turtle has 2 constructors…

                  One is takes no formal parameters and constructs a default Turtle

Called a Default (no-arg) Constructor

                  One takes 4 formal parameters (i.e. arguments):

                                    Called a 4 Arg Constructor

                  The class also has a number of Accessors and Mutators.

                  The isPenDown method

                           Returns true if the pen is down and false if it is up.

                          

An interface specification like this is similar to what programmers might be given in a professional setting.

Software engineer decides what all of the classes are and how they will interact. Then each programmer implements and tests one or more of the classes independently.  Then bring them all back together to make the final application.

 

The interface tells us a lot, but not everything...

         It doesn’t tell us how to write the source code that implements those methods.

         It also doesn’t tell us exactly what fields the class contains.

                  Though the accessor methods may give us good clues.

 

So given a specification the first thing a programmer (that’s you!) will need to do is decide what fields (i.e. state) the object might need.

 

So what field might a Turtle object need to have?

Write the code for the fields:

 

private double xPos;

private double yPos;

private int angle;

private String penColor;

private boolean penDown;  // true means pen is down.

 

Write the code for the default constructor:

 

public Turtle() {  

xPos = 50;

yPos = 50;

angle = 0;

penColor = "black";

penDown = false;

}

 

Write the code for the 4 arg constructor:

 

public Turtle(double initX, double initY,

int initAngle, String initColor) {

 

xPos=initX;

yPos=initY;

angle=initAngle;

penColor = initColor;

penDown = false;

}
