CoinCollection and VideoStore examples all used an ArrayList to implement a collection of objects.

         Two features of these examples that are worth noting:

                  We didn’t need to know the size of the collection in advance.

                  The size of the collection could change over time.

 

         ArrayList:

Useful for variable size collections.

Stores object references.

 

Some collections are different:

         We may know the size of the collection in advance and that size may also be fixed.

                  e.g.    the number of hours in a day

                           a list of the US Senators

                           a list of stock prices for the last 20 days

the number of quiz or homework grades for each student in a particular class

 

We could of course implement these types of collections using an ArrayList too.

However, because the ArrayList has the ability to change size its code contains a lot of overhead that is not needed in cases where we know the size of the collection and that size is fixed.

 

         Today and the next few classes we’ll be working with something called an:

 Array: Used for fixed size collections.

        

The fact that arrays are fixed size may have limitations but they also have

Two benefits:

Store either object references or primitive values.

                  Accessing elements is more efficient than with ArrayList

 

Our examples:

We’ll develop a program for analyzing a fixed number of days worth of stock prices.

 

Declaring an Array Variable:

double[] prices;

 

Creates an array variable that can refer to an array that can hold double values.

         The [] make this different than declaring a variable of type double.

                           e.g. double price;  // not an array.

 

         [] in the declaration indicate that prices is an array variable.

 

Declaring an array variable doesn’t create an array.

It only creates a variable that can hold a reference to an array.

         In the same way declaring an object variable doesn’t create an object.

Array variables have the default value null, just like an object variable.

Using Array Objects:

        

         Getting the value of an array element:

                  double price = prices[2];

 

         Setting the value of an array element:

                  prices[1] = 78.33;

                 

Each element of the array acts like an independent variable of the specified type.

         So we can do things like:

                  prices[0]  = 2*prices[1] + prices[2];

 

         Finding the length of an array:

                  int len = prices.length;

 

Notice the difference from ArrayLists.  With an ArrayList we called the .size() method to find out how many things were in the ArrayList.  With an array we use .length which is a field, not a method, thus no () after .length.

public double getPrice(int day) {

if (day >= 0 && day < prices.length) {

return prices[day];

}

else {

System.out.println("Error: Invalid day.");

return -1;

}

}

        

Why return the value –1 when the day is invalid?

If the day is not a valid one, then there is no actual price to return.  So we want to return something that could not be mistaken for an actual price.
