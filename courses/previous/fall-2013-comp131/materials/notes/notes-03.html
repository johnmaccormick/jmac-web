We could think about a program for this machine as a sequence of knob & switch settings.  That is a program would just be the specifications for how to set the knobs and switches for each CPU cycle.  The settings for a single CPU cycle are called an instruction.

 

         For example: A program that computes R3=R2+R1-R0 might be written as:

                          

                  Cycle  Abus  Bbus  ALU  Switches  Cbus   Memory

                    0         2         1        +         COOC      3          0         R3¬R2+R1

                    1         3         0        -          COOC      3          0         R3¬R3-R0

 

                  Switches are coded C or O starting at top with CBus switch.

                  Irrelevant values can be set to zero.

 

         What would the instruction look like if we wanted to compute MM[11]¬2*R3?

        

                  Cycle  Abus  Bbus  ALU  Switches  Cbus  Memory

                    3         3         3        +         OCOC      0          11      MM[11]¬2*R3

Only problem is that this version of the K&S, is representative of a First Generation machine and it can only be programmed in Machine Language which consists of just 1’s and 0’s…

 

So we need a way to represent those instructions that we came up with using only 1’s and 0’s.

Switches:

         Represent closed switches with a 1.

         Represent open switches with a 0.

Now for the knob positions, we need a way to represent 0, 1, 2, 3 and 11 using just 1’s and 0’s.

 

The numbers we are used to are:

Decimal Numbers (base 10):

                  Digits 0-9.

 

Computers use:

Binary Numbers (base 2):

                  Digits 0-1.

                  Each digit is called a bit.

                  The 0 and 1 roughly correspond to an open or closed switch.

 

In the decimal numbers that we are used to each digit represents a different power of 10:

 

         342            3                4                2

                           102             101             100

                           hundred’s ten’s          one’s

        

         or 3*102 + 4*101 + 2*100

         or 3*100 + 4*10 + 2+1 = 342

 

Binary numbers work in an analogous way, except they only use the digits 0 and 1 and they use powers of 2.

 

         101            1                0                1

                           22               21               20

                       four’s         two’s          one’s

 

To find out what base 10 value this binary number corresponds to we can interpret it the same way we interpreted the base 10 number above – but using a base of 2!

 

                  1*22 + 0*21 + 1*20

                  1*4 + 0*2 + 1*1 = 4 + 1 = 5.

 

Another Example:

         1       1       0       0       1

         24      23      22      21      20

         16     8       4       2       1

 

         16+   8+                       1       = 25.

 

 

Ex:    Convert a decimal to binary

         List out the powers of 2 until you reach the last one not greater than 11.

        

         __     __     __     __

23      22      21      20

8       4       2       1

 

There is 1 eight in 11 so put a 1 in the 8’s column.    11 – 8 leaves 3.

There are 0 fours in 3 so put a 0 in the 4’s column.

There is 1 two in 3 so put a 1 in the 2’s column.      3 – 2 leaves 1

There is 1 one in 1 so put a 1 in the 1’s column.      1 – 1 leaves 0

 

 1      0       1       1

23      22      21      20

8       4       2       1

 

Check:               8 + 2 + 1 = 11.
