Interesting questions.

         Are there any break-even days in the data?

          In other words are there any duplicate values in the data?

 If so then it is possible to buy and sell the stock during the current period and break even.

 

What is the maximum profit per share that could have been made during the period using a single buy and a single sell?

 

How much money would we make if we trade using the following strategy:  Buy a stock the first time the price increases, then hold it until the next time the price decreases?

 

Let’s consider the problem of finding out if there are any break-even days (i.e. duplicate values) in the data.

 

         Draw an object diagram of an StockAnalyzer object with prices field.

                  27 23 28 15 9 6 12 23 19 10

 

         How can we determine if there is a duplicate in the data?

                 

         Present this idea:

For each entry in the array, go through the rest of the entries in the array and see if there is a match.

 

For 27, look at 23, 28, 15, 9... etc

There is no match so move onto the next element.

For 23, look at 28, 15, 9, 6, 12, 23

There is a match so we are finished – the list contains duplicates.

 

                  What happens if there is not a duplicate?

Eventually we would compare the 19 to the 10 and at that point, if we haven’t found a duplicate then we can be sure that there are not any duplicates in the data.

 

                  Some other good questions to ask:

                  Does it work if the first two numbers are the same?

                           Does it work if the last two numbers are the same?

 

 

 

                          

So let’s translate that idea into Java code.

 

/**

 * Determine if there are any pairs of days on which buying the

 * stock on one day and selling it on the other would lead to

 * breaking even.  In other words, are there two days that have

 * the same closing price. If so, return true, otherwise return

 * false.

 *

 * @return true if there are break-even days and false if not.

 */

public boolean hasBreakEvenDays() {

     for (int i=0; i<prices.length-1; i++) {

         for (int j=i+1; j<prices.length; j++) {

              if (prices[i] == prices[j]) {

                  return true;

              }

         }

     }

    

     return false;

}

 

i indicates the index of the value for which we are going to look for a duplicate.

 

We use i < prices.length-1 because we don’t need to look for a duplicate of the last element.

 

j is the index of the element that we are currently checking to see if it is a duplicate of the element at index i.

 

i and j are commonly used as indices for for loops that iterate over arrays when there are not other sensible names to be used.
