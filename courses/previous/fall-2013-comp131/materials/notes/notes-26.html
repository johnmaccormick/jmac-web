We’ve been doing lots of examples where we have needed to iterate over all of the objects in an ArrayList.  To do so we’ve used a for loop.

 

This has been great, but there are some things that we might want to do that are either not easily done using a for loop or may be inefficient when using the for loop.

 

For example consider the problem of:

         Finding the first overdue DVD in the collection.

         Getting a collection of the first 5 overdue movies.

 

The issue is that we don’t know how many of the DVDs we will have to go through to find the first one that is overdue.  Maybe just one. Maybe all of them.

 

So what we need is something that will allow us to repeat statements over and over again like the for loop but that can terminate as soon as we are finished with what we are trying to do.

 

One such tool is the while loop:

         while (<loop condition>) {

         <loop body>

     }

 

A while loop executes the statements in the loop body over and over while the loop condition is true.

 

         Precisely what happens is this:

When program execution reaches a while loop, the loop condition is checked.

 

If the condition is true, then the statements in the loop body are executed. When all of the statements in the loop body have executed, execution jumps back to the top of the loop and the loop condition is checked again…

 

If the condition is still true the process is repeated.

 

If the condition is false, the body of the while loop is skipped and execution continues with whatever statements follow the loop body.

public DVD getFirstOverdueDVD() {

DVD overdue = null;

int index = 0;

while (index < dvdList.size() && overdue == null) {

DVD dvd = dvdList.get(index);

if (dvd.isOverdue() == true) {

overdue = dvd;

}

else {

index++;

}

return overdue;

}
