Homework Assignment 6d

  1. Consider the following constructor definition (from the BankCustomer class with fields name, address, phoneNum, checking, and savings): 
        public BankCustomer(String initName, String initAddress, String initPhoneNum,
                        Account initChecking, Account initSavings)
    {
        name = initName;
        address = initAddress;
        phoneNum = initPhoneNum;
        checking = initChecking;
        savings = initSavings;
    }
    Suppose that the formal parameters of the constructors are changed to the following: 
        public BankCustomer(String name, String address, String phoneNum,
                        Account initChecking, Account initSavings)
    How can you rewrite the constructor definitions so that the constructors will still work correctly (with these formal parameter names)? (Hint: use this.)
  2. Add a method to the BankCustomer class to transfer money from one customer's checking account to a different customer's checking account. The signature of this method must be: 
    public void transfer(BankCustomer fromCustomer, int amount)
    In other words, the method takes a BankCustomer object and an amount to transfer as parameters, and transfers money from the checking account of the BankCustomer object passed as a parameter to the checking account of the BankCustomer object used as the calling object for the method. Your method must display an error message if sufficient funds are not available for the transfer, and not change any account balances in this case. Copy and paste your method definition as your answer for this exercise.

    Hint: it may be helpful to add an accessor method for the checking account balance to the BankCustomer class, and then use that method in implementing the transfer method. You may have already implemented this accessor in a previous practice exercise.
