Constructor:

A constructor is a special kind of method that you call to create a new object.

                  For example: new Circle()

 

These methods were invoked by choosing them from the Class instead of on a specific object.

 

State:

In today’s exercises you’ll be exploring something called Object State or simply State.

 

The state of an object is information that the object uses to describe the thing that it is representing.

 

For example each Circle object contains information about the circle such as:

         diameter

color

x position

y position

 

Each Circle has its own unique values for each of these pieces of information.

         i.e. each Circle has its own state.

One circle object can represent a blue circle with a diameter of 10 and another can represent a red circle with a diameter of 40.

        

The class Circle says that all instances of the Circle class will have these attributes (also called fields),

Each instance of the Circle class has specific values for the attributes that make it unique.

 

What state might an object modeling a BankAccount have stored inside it?

         AcctNumber, Balance

         Each BankAccount object will have its own AcctNumber, Balance

 

 

What about an object modeling a Student?

         Name, IDNumber, Schedule

         Each Student object has its own Name, IDNumber, Schedule

 

Recall that a method is an action that can be performed by an object. 

         For Example:

A Circle had methods:

                           changeColor that changed its color.

                           changeSize that changed its size.

 

What methods like these do is that they change the state of the object on which they are called:

 

         For Example:

The changeColor method changes the part of a Circle’s state that represents its color.

 

The changeSize method changes the part of a Circle’s state that represents its diameter.

Recap:

         What are Fields?

                  Fields are what objects use to hold their state.

                  Fields = Attributes

        

The Class determines:

The fields that each object belonging to that class will have.

And also the data type of each field.

 

The methods that each object belonging to that will class have.

And also exactly what the methods will do when called on an object.

        

The Source Code of the class determines these things.

 

Each object instance maintains specific values for its fields (i.e. its state).

When we invoke a method on an object, that method manipulated the fields of that object instance.

 

E.g. When you call moveHorizontal with a parameter of 10 on a specific Circle object, it is the x position of that Circle object that is increased by 10.

 

 

For Exercise, 1.14, you added (or will add) a second sun to the picture by declaring a new field:

 

private Circle sun2;

 

and then writing code like:

 

sun2 = new Circle();

sun2.changeColor("yellow");

sun2.makeVisible();

 

in the draw method. 

 

The line:

 

sun2 = new Circle();

 

creates a Circle object – just like right clicking on the Circle class in BlueJ, except that the program now automatically creates the object.

 

The line:

 

sun2.changeColor("yellow");

 

calls the changeColor method and passes it the parameter "yellow" – just like right clicking on a Circle object on the object bench in BlueJ and calling the changeColor method from the menu, except that the program does it automatically!
