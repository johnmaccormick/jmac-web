When did you need to compile?

Every time you make a change to the source code, you need to compile before running.

 

How did you compile the class?

Best bet is to click the “Compile” button in the main BlueJ window.

Can also click “Compile” in the source code window but that only compiles that one Class.

 

What does compiling do?

It translates the Java source code into a “kind of” machine language called Java Byte Code.

 

This Java Byte code is then interpreted by the Java Virtual Machine (JVM) which is a program that simulates a computer… much the same way the Knob and Switch simulated a computer – but way more sophisticated.

 

What are fields / attributes / Instance variables?

         They are the information holders that an object maintains about the thing it is representing.

 

What is an object’s state?

         The collective values of all of the fields of an object make up its state.

        

How is an object’s class related to its state?

The class determines what fields an object has.  All objects belonging to a class have the same fields.  Each object instance has its own values for those fields (i.e. its own state).

 

What are the Java Data Types that we’ve seen so far?

One more we’ll see as we go along:       double

Data types are a very important idea in Java programming:

Where are these data types used?

Every field has a data type that determines the types of values that it can hold.

Every parameter has a data type that determines the types of values that can be used when calling the method.

In fact, every place that we use a value in a program the type of that value will be precisely specified.

 

Those data types above are built in to Java.    In addition...

Every class defines a new data type.

                  Circle is a data type.

                  Car is a data type.

                 

         Because every class defines a data type we can say that:

An object is a value of its class’ data type.

         Just another way of saying that an object is an instance of its class.

 

                  An analogy might help:

7 is to int as sun is to Circle.       

 

                  How about one for String and Car?

                           “Bob” is to String as my Pathfinder is to Car.

 

So if a class defines a data type and a data type is used to define the types of values that a field can hold then:

                  Objects can have fields whose data type is defined by a class.

                           Such a field will have an object as its value.            

The Picture class had a field of type Circle for the sun.

Not only can data types defined by classes be used for fields, they can be used anywhere that a data type is required.  For example...

                  When calling such a method you will pass an object as the parameter.

What can the methods of LabClass do?

There are two ways to find out in BlueJ

 

You can create an object of type LabClass and right click it to get the list of methods that can be called.

                          

Can also open source code file and look at the Documentation View.

This view shows a list of all of the methods and constructors in the class along with information about what each of the methods does and the parameters that it takes.

 

If you are just using a class and not changing its source code, this view is much more useful than looking at the source code itself. It shows you only what you need to know.

 

The things displayed in the interface view are the Method Signatures:

 

         For example:

         void enrollStudent(Student newStudent)

 

                  The signature indicates 3 important things about a method:

                           Name

                           Parameters (data type followed by name)

                             If there are multiple parameters they are separated by commas.

                           Return type

                          

So just by looking at the method signatures we can tell a lot about the methods and we can learn exactly how to use them.

Return Values:

All of the methods that we have called so far did something to the object when they were called.  E.g. makeVisible(), moveHorizontal(), slowMoveVertical() etc…

 

There are also methods that will tell us something about the object when they are called.  These methods return a value to us indicating something about the object.

 

We can tell when a method has a return value by looking at its method signature:

                  If the Return Type is:

                           void – the method does not return a value.

                           A data type – the method returns a value of that type.

                          

         For example: String getName()

This method returns a String value telling us the Student’s name.

Recap

There is an important distinction between information printed in the console and information that is returned from a method call.

We will be able to use information that is returned from a method call at other places in our program.  For example, imagine we want to know the total number of credits taken by all students.  What could we do?

 

         If we print each student then we can add up the credits by hand.

It will turn out that there is an easier way.  We will be able to write code that will make use of the return values from the getCredits method to sum up the credits for each student.
