Testing by hand:

         Create a Turtle object on the object bench.

         Call its accessor methods

getX, getY, getAngle, getColor, isPenDown

Check each value with what is expected...

 

         As you can see this can get a bit tedious….

Also if you were to find a bug in the code and attempt to fix it,
then you’ll need to run all of your tests over again.

First, to be sure that you fixed the bug

Second, to be sure that the changes you made didn’t break anything else.

 

Automated Testing:   (This is not in the text!)

 

To help with testing BlueJ has built in support for an automatic testing system called JUnit.

 

The idea is that you will go through your test once and BlueJ will record it.

Then you’ll be able to re-run your test automatically, as many times as you need to.

 

         Creating a test class  (enabled?)

         Right click Turtle class

                  Choose “Create Test Class”

        

Recording a test case for the default constructor.

         Right click TurtleTest

         Choose “Create Test Method”

                  Make test for testDefaultConstructor

Convention: Test methods are named testXXXXX

 

Recording indicator comes on.

Everything you do now will be recorded into the testDefaultConstructor
method and saved so that you can re-run it later.
