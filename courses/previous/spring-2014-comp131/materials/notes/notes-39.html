Topic #39: The main Method

 

Thus far this semester every program you have worked with has relied on BlueJ in order to be able to run.

 

Today we’ll see how to create a Java program that can run without the help of BlueJ.

 

First, let’s look at a program we’ve used before and see what we have to do to get it to run within BlueJ.

         (Look at the StockAnalyzer class)

 

         To get this program to run within BlueJ we created a StockAnalyzer object.

         Then we invoked the fetchClosingPrices method.

         Then we would invoke a variety of the other methods that we wanted to use.

                  (e.g. buyOnFirstIncrease)

 

We can run this program without using BlueJ by creating a main method:

The main method a special method that is automatically invoked when a Java program is run without the benefit of BlueJ.

 

         A main method has the following method signature:     

(enter this code into the StockAnalyzer project)

 

     public static void main(String[] args) {

 

     }      

 

public means that the method can be used outside the class.

static means that the method is associated with the class and not a specific object.

This is important because when the program starts, there is no object on which we could call the method.  But the class does exist.

String[] args – an array of String objects.

         More on this in a moment.

 

The signature of the main method is always exactly the same!

 

         Any code placed in main will be executed when this Java program is run.

 

         So what we want to do in main is the same kind of thing that we did in BlueJ:

         // Create an object of type StockAnalyzer.

         // Invoke the fetchPrices method

         // Invoke analysis methods.        


            // Create an object of type StockAnalyzer.

          StockAnalyzer stock = new StockAnalyzer(“GOOG”, 20);

 

          // Invoke the fetchPrices method

          stock.fetchClosingPrices(2006, 4, 1);

 

          // Invoke analysis methods.   

          double profit = stock.buyOnFirstIncrease ();

          System.out.println(“Profit for GOOG: “ + profit);

Command Line Arguments:

 

It would probably be more convenient if we didn’t have to recompile every time we want to change the stock or the number of days or the date. It would be much more convenient if we could just tell the program which stock to use and how many days to use when we run the program.

 

         Something like:      java StockAnalyzer GOOG 20 2006 4 1

 

                  This is where that array of Strings comes into play.

Each thing listed after the class name gets turned into a String and placed in successive locations in the array of Strings referred to by args.

These values are called Command Line Arguments

        

So we can change the code to use the command line arguments when constructing the new StockAnalyzer object.

 

                  // Create an object of type StockAnalyzer.

          String symbol = args[0];

          int days = ????

          int year = ????

          int month = ????

          int day = ????

 

     StockAnalyzer stock = new StockAnalyzer(symbol, days);

                  stock.fetchClosingPrices(year, month, day);

 

         Note that args[1] holds the number of days.

         Similarly, args[2], 3, 4 hold the year, month and day.

 

However, they are held as Strings and within the StockAnalyzer we need them to be int values.

For example the number of days is “20” and the year is “2006” which remember are Strings which are different than the int values 20 and 2006.    

 

The Integer Class:     (Look at javadoc)

 

The Integer class has a number of constructors – one accepts a String containing an integer number as a parameter.

 

One method in the Integer class is intValue() which returns the value as an int.

                          

         Integer daysInt = new Integer(args[1]);

          int days = daysInt.intValue();

Integer yearInt = new Integer(args[2]);

          int year = yearInt.intValue();

Integer monthInt = new Integer(args[3]);

          int month = monthInt.intValue();

Integer dayInt = new Integer(args[4]);

          int day = dayInt.intValue();
