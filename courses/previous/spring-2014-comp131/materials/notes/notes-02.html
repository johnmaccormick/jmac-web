Topic #2: How Computers Work

 

Stored Program Architecture idea came from John Von Neumann

 

The idea of the stored program architecture was that:

Both the program and the data on which it is to operate were stored in the computer’s memory.

 

By putting the program in the computer’s memory it could be run over and over again very rapidly and more reliably

 

The stored program architecture is made up of three main components:

         Central Processing Unit (CPU)

         Memory

         Input/Output (I/O) Devices

 

CPU: Executes the instructions that make up the program.

Memory: stores the running program and the data on which it operates.

I/O Devices: Allow user interaction with the machine.

         Back in the day this was paper tapes / punch cards / blinking lights / printer

         Now it includes keyboard / mouse / monitor / network / printer

 

The idea is:

         CPU reads a program and data from an I/O device and stores it in the memory.

Then it retrieves the instructions from memory (one-by-one) and executes each one.  So the CPU is where all of the processing takes place!

 

The CPU is also made up of 3 main components:

         Arithmetic Logic Unit (ALU)

         Control Unit

         Registers

 

The ALU performs the mathematical and logical operations indicated by the program instructions.

The registers hold the values that are being used in those mathematical operations.

The control unit is “the conductor” of the computer

It is the part that retrieves the instructions, figures out what they are, and makes all of the parts of the computer perform the operation corresponding to the instruction.

 

To make all of this a little more concrete, we will be working with a simulation of a stored program architecture computer.  This simulation will give you an idea of how data moves around inside the computer and how instructions get executed.  By the end you should have some feeling for how computers are able to do what they do using only 1’s and 0’s.

 

Knob and Switch Computer - Simulation Version 1:

         This simulator shows us 2 of the 3 parts of the CPU.

                  The Registers (4 of them) and the ALU

                  The Registers and the ALU are connected by Busses

Busses allow data to move from the registers to the ALU for processing and back to the registers.  In a real computer Busses are simply a collection of wired along which data and instructions move.

 

                  For now, we will be playing the role of the control unit.

                           As we go along we’ll be adding parts to this simulation

                           Eventually we’ll get a full-blown stored program architecture machine.

 

         Take a look at our ALU:

                  It takes 2 inputs (A and B) that come from the registers

                  It can perform 4 operations:

                  A+B, A-B, along with a logical AND and logical OR operation

                           We’ll just be concerned with A+B, A-B

                           Notice no division or multiplication - this is a simple computer.

 

         There are 4 registers (R0-R3 for register 0 to register 3):

                  Computer scientists usually start counting from 0.

 

                  Each register can store a single data value.

                          

                  The knobs control what the computer is going to do

                           A Bus Knob: Selects the register whose value is put onto the A Bus.

                                    Click it to change its setting.

                           B Bus Knob: Selects the register whose value is put onto the B Bus.

The Knob in the ALU selects what operation will be performed on the data.

C Bus Knob: Selects the register where the result of the ALU operation will be stored.

 

Example: R3 = R0 + R2

Add R0 to R2 and store result in R3.

 

As you may know the speed of computers today is measured in Giga Hertz.

1 Hertz = 1 clock cycle per second. 

Modern computers are able to perform one simple operation such as addition in one clock cycle.  So the CPU execute cycle we looked at can be done in one clock cycle.

So a 1 Giga Hertz machine can perform 109 (1 billion) CPU cycles per second.
