Recap:    What is composition?

         Composition: Using objects as fields in other objects.

That is having the data type of a field in a class be another class.

E.g. The BankCustomer

Today we’ll talk about two types of diagrams that are useful for creating and understanding programs that use composition.

 

         Class Diagram: Illustrates relationships between classes in a program.

Object Diagram: Illustrates relationships between objects as a program is running.

 

Class Diagram:

 

         There is a box for each class, with the name of the class at the top.

                 

If Class A has a field of type Class B then there is an arrow from Class A to Class B.     



 

 

 

 

 

 


So to draw a class diagram.  Just draw one box for each class. Then for each class, draw an arrow to any class that it uses as the data type for a field.

Arrow is called a uses arrow.

 

This is what is shown in the BlueJ window.

        

NOTE: It turns out that String is actually a Class in Java. So we should really draw a uses arrow to a box for the String class from each of these classes.  However, Strings are used so often that they are treated a little differently than other classes and objects, so we will not be doing that.

Each arrow is a Reference:

- indicates the location of an object in memory

- a variable whose data type is a class will hold a reference to an object of that type.

                  They are the same objects as on the bench.

 

When we create an object and give it a name, such as sAcct or cAcct what that computer is doing is placing into a variable with that name, a reference that tells it where in memory to find the actual object.

Two categories of data types:                          

Primitive Data Types

Object Data Types

 

         Primitive Data Types:

                  int, double, boolean (and others)

                  built into java

                  not defined by classes

                  primitive type variables hold the actual value

NOTE: A variable is just a storage location in memory. In java variables include local variables, fields and also formal parameters.

 

         Object Data Types:

                  defined by classes

                           Account, Address, Student, (String)

                           Some are defined by us: e.g. Account

                           Others come packaged with Java:  e.g. String

                                    We’ll see lots more of these as the semester progresses.

                  object type variables hold a reference to the object

                           reference = memory address
