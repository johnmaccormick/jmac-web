Recap - Distinction between a return value and printing information:

 

There is an important distinction between information printed in the console (terminal) and information that is returned from a method call (to another part of the program).

 

For example, imagine we want to know the total number of credits taken by all students.  What could we do?

 

If we print each student then we can add up the credits by hand. Fine if we have 10 or 20 students but not 100’s or 1000’s or if we want to do it frequently.

 

It will turn out that there is an easier way.  If we return the value from a method instead of printing it, then other code will be able to process that value – e.g. adding up the credits with program code.

Stuff after // on a line are  comments

Comments are ignored by the computer but are included to explain your code to a human reader.  Programmers write comments to help other (???)  programmers understand how their programs work.  This is important because programs are rarely static – instead they are constantly being fixed, updated and extended – often not by the original programmer.  So, good comments can save a substantial amount of time because the new programmer will need to spend less time figuring out exactly how the program works.

 

For example: // in pennies

This makes it clear that the balance of the Account will be stored in cents and not dollars and cents.  This is not obvious from the name of the field but will be important for understanding the rest of the code in the class.

 

Stuff following /* and before */ are also comments.    If we use /** and */ that is a special type of comment called a Javadoc comment.  You’ll notice that Javadoc comments appear at the top of the class and also before every constructor definition and every method definition. They give an overall description of the class, method or constructor and will be used to automatically generate external documentation.

Assignment Statement

Assignment statements are used to assign values to fields (& other variables).

 

balance = 0;

<variable> = <expression>;

Expressions can be a literal value as was the case here.  They can involve formal parameters, variables, arithmetic operations (addition, subtraction etc.) and even method calls.

 

What goes on the right hand side of an assignment statement is an expression and has a value.

 

         We say --  The value 0 is assigned to balance.

                  This is somewhat like a formula but not like an algebraic equation.

 

The value 0 is being copied into the memory cell for balance.

 

Type Checking

This means that the value of the expression on the right hand side of the assignment must have the same data type as the value of the field on the left hand side.

 

         E.g. 0 is of type int and the balance field has type int.

 

                  balance = “One Million”;

                           This is not valid!

 

Similarly, when you give a value to a parameter, the data type of that value must match the declared type of that parameter.

More about parameters.

 

Turns out there are 2 types of parameters.  The one we’d been talking about is called the:

 

         Actual Parameter (a.k.a.  Argument)

This is the value that you provide for a parameter when you create an object or call a method. 

E.g.   1234 for acct when creating an Account.

         50 when calling moveHorizontal on a Circle

        

         Formal Parameter

                  The variable that appears in the constructor or method definition.

                  E.g.   initAcctNumber in the constructor of the Account class.

 

Formal parameter names given in the header of a method definition can only be used in the code for that method. 

 

This is an example of something called scope.

The scope of a name is the portion of the source code in which that name can be validly referenced.

 

         The scope of a private field is its class definition.

E.g.   acctNumber can be used anywhere in the definition of the Account class.  (i.e. between the matching set of curly braces for the class).
