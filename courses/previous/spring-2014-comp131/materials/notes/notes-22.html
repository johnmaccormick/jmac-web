Testing addDVD and getDVD:

        

The general plan is to add some DVDs, then check to see if the collection size is correct and also check that the added DVDs are in the correct spots.  We could check them all but that’s probably overkill.  We can assume the add method works correctly, so we really only need to verify that we are calling it correctly.  So we’ll just check the first and last elements, if those are in the right places then it is safe for us to assume that the rest are too.

 

Testing Rule: Methods that add to a Collection:

Add several elements

Check the size of the collection

Check the first and last elements

Check just enough fields

Add several elements

Remove an element from the middle of the list

Check the size of the collection

Check the element that is now at the removed index

Check just enough fields

How should we test printDVDTitle?

This method doesn’t return a value (like an accessor) or change the state of the object (like a mutator).

 

         So the only way to test it is to invoke it and then manually inspect the output.

So the thing to do for this method (and any similar method), is just to record a call to it in a test method.  Then when you run your tests, the output will be generated and you can inspect it.

 

Good idea to always clear the terminal window before running the test cases.
